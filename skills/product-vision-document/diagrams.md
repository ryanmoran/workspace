## Tips for Creating Diagrams

### Do:
- Use simple shapes (boxes, arrows)
- Label everything clearly
- Show the user's perspective
- Include start and end states
- Use emoji sparingly for visual interest (ğŸ“± ğŸ‘¤ âœ“ âœ—)

### Don't:
- Show technical architecture (databases, servers, APIs)
- Use technical abbreviations without explanation
- Create diagrams with more than 10 elements
- Assume the reader knows system terminology

### Accessibility:
- Use high contrast (solid lines, not dotted)
- Don't rely on color alone to convey meaning
- Include text descriptions alongside diagrams
- Test that diagrams make sense without any color
